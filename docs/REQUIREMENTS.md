# 요구사항과 완료 기준

우선순위는 `MUST`, `SHOULD`, `COULD`로 표시한다.

## 기능 요구사항

| ID | 우선순위 | 요구사항 | 완료 기준 |
|---|---|---|---|
| FR-01 | MUST | 부품을 일정 간격으로 생성한다. | 설정된 개수와 간격이 로그와 일치한다. |
| FR-02 | MUST | 컨베이어가 부품을 검사 위치까지 운반한다. | 이탈·관통 없이 센서 구간을 통과한다. |
| FR-03 | MUST | 센서 이벤트로 검사 순서를 시작한다. | 한 부품당 감지 이벤트가 정확히 한 번 발생한다. |
| FR-04 | MUST | 검사 중 컨베이어를 정지하고 결과를 만든다. | 결과가 PASS 또는 FAIL 중 하나이며 타임아웃되지 않는다. |
| FR-05 | MUST | FAIL 부품을 로봇이 불량 박스로 옮긴다. | 부품이 박스 영역 안에 놓이고 집계가 갱신된다. |
| FR-06 | MUST | 생산 KPI를 집계한다. | `total = pass + fail` 불변식이 항상 유지된다. |
| FR-07 | MUST | 장애 시 라인을 안전 정지한다. | ERROR 상태에서 컨베이어/로봇 명령이 중단된다. |
| FR-08 | MUST | Reset 후 초기 운전 상태로 복귀한다. | 오류 원인 해제 후 다음 부품을 처리한다. |
| FR-09 | SHOULD | 운전 시나리오를 설정 파일로 바꾼다. | 코드 수정 없이 3개 속도를 실행한다. |
| FR-10 | SHOULD | 실행 결과를 JSON/CSV로 저장한다. | 시나리오별 KPI와 설정을 추적할 수 있다. |
| FR-11 | COULD | 규칙 기반 시각 결함을 판정한다. | 결함 Ground Truth와 판정 결과를 비교한다. |

## 비기능 요구사항

| ID | 범주 | 요구사항 |
|---|---|---|
| NFR-01 | 재현성 | 모든 확률 동작은 seed를 설정할 수 있어야 한다. |
| NFR-02 | 구조 | 배치·물리·동작·재질·조명을 독립 USD 레이어로 관리한다. |
| NFR-03 | 테스트 | 상태 머신과 KPI는 Isaac Sim 없이 단위 테스트할 수 있어야 한다. |
| NFR-04 | 성능 | 목표 장비에서 안정적인 실시간 시연 프레임을 유지한다. 목표값은 1주차에 측정 후 확정한다. |
| NFR-05 | 관찰성 | 상태 변경, 판정, 장애, Reset에는 시간과 부품 ID가 기록되어야 한다. |
| NFR-06 | 안전성 | 오류 상태의 기본 동작은 정지이며 자동 재가동하지 않는다. |
