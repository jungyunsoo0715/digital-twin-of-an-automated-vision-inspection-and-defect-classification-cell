# 프로젝트 정의서

## 1. 목적

산업 부품의 투입, 감지, 검사, 정상 통과, 불량 분류 과정을 재현하는 소규모 생산 셀 디지털 트윈을 구축한다. 시각적 장면에 그치지 않고 설비 상태, 생산 KPI, 장애, 운전 조건 변경 결과를 설명할 수 있는 포트폴리오를 만든다.

## 2. 핵심 질문

1. OpenUSD로 장비 에셋을 재사용 가능하게 구성할 수 있는가?
2. 센서·컨베이어·검사·로봇을 하나의 상태 머신으로 안정적으로 제어할 수 있는가?
3. 컨베이어 속도가 증가할 때 생산량은 계속 증가하는가, 아니면 검사/로봇 병목이 발생하는가?
4. 장애를 감지하고 정지·알림·복구 흐름을 재현할 수 있는가?

## 3. 이해관계자 관점

| 관점 | 필요한 결과 |
|---|---|
| 생산 관리자 | 생산량, 불량률, 사이클타임, 정지 원인 |
| 설비 엔지니어 | 상태 전이, 센서 이벤트, Reset, 재현 가능한 장애 |
| 시뮬레이션 개발자 | 레이어 분리, 에셋 재사용, 테스트 가능한 코드 |
| 포트폴리오 평가자 | 2~3분 내 이해되는 시연과 정량 비교 |

## 4. 성공 기준

- 30개 이상 부품이 연속 처리되는 시연에서 상태 머신이 교착되지 않는다.
- PASS/FAIL 집계가 처리된 부품 수와 항상 일치한다.
- 부품 걸림 또는 픽업 실패를 강제로 재현하고 Reset 후 재가동할 수 있다.
- 0.2, 0.35, 0.5 m/s 시나리오를 동일 조건으로 실행해 결과를 비교한다.
- 외부 에셋 원본을 수정하지 않고 레이어/Reference 구조를 문서로 설명한다.

## 5. 범위

### 반드시 완성

- 컨베이어 이동, 센서 감지, 검사 위치 정지
- 확률 또는 규칙 기반 PASS/FAIL 판정
- 로봇의 고정 Pick/Place 불량 제거
- KPI와 설비 상태 표시
- 장애 1개 이상, Reset, 로그
- OpenUSD 구성과 실행 절차 문서

### 여유 시 확장

- 색상·크기·결함 마커 기반 판정
- 제품 Variant, MQTT/REST, ROS 2
- 웹 대시보드와 결과 그래프 자동 생성

### 제외

- 실제 비전 모델 훈련, 실제 PLC, 다중 로봇
- 공장 전체, 복잡한 경로 계획, 용접/절삭 물리
