# 시험 및 검증 계획

## 시험 계층

1. 단위 시험: 상태 전이, 판정 재현성, KPI 불변식
2. 통합 시험: 센서 → 정지 → 검사 → 로봇/배출 이벤트 연결
3. 물리 시험: 제품 안정 이동, 충돌, 정지 위치 오차
4. 시나리오 시험: 정상 운전, 걸림, 픽업 실패, Reset
5. 성능 시험: 속도별 처리량과 병목, 렌더/물리 성능

## 필수 시험 사례

| ID | 시나리오 | 기대 결과 |
|---|---|---|
| TC-01 | PASS 강제 판정 | 정상 수량 1, 총수량 1, 컨베이어 재가동 |
| TC-02 | FAIL 강제 판정 | 불량 수량 1, 로봇 작업 1, 컨베이어 재가동 |
| TC-03 | 정의되지 않은 상태 전이 | 오류를 즉시 발생시키고 상태를 유지 |
| TC-04 | 부품 걸림 강제 발생 | ERROR, 생산 수량 미집계, 마지막 장애 기록 |
| TC-05 | 픽업 실패 강제 발생 | ERROR, 불량 처리 미완료, Reset 요구 |
| TC-06 | Reset | IDLE을 거쳐 정상 운전 가능 |
| TC-07 | 같은 seed로 2회 실행 | PASS/FAIL 순서와 KPI가 일치 |
| TC-08 | 100개 처리 | `total = pass + fail`, 음수/NaN 없음 |

## What-if 실험 통제

- 변경 변수: 컨베이어 속도만 0.2, 0.35, 0.5 m/s로 변경
- 고정 변수: 부품 수, 결함률, seed, 검사시간, 로봇 사이클타임
- 반복: 속도별 최소 3회, seed 목록을 동일하게 사용
- 측정: 시간당 처리량, 평균/95백분위 사이클타임, 로봇 대기, 장애 수
- 해석: 처리량 증가가 검사/로봇 용량을 넘을 때 큐 또는 대기시간이 증가하는지 확인
